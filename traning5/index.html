LGWAN環境 Webアプリ開発研修

最終課題：消防水利マップの開発

オープンデータを活用した実践的Webアプリ構築

本日の目標

  • これまでの研修の集大成として、「志賀町 消防水利マップ」を各自で作成します。
  • 志賀町のオープンデータ（消防水利一覧）を実際に読み込んで利用します。
  • 有事の際、町民や県外からの応援支援者（消防関係者）が、スマートフォンで直感的に使えるアプリを目指します。

課題の要件定義

以下の基本要件を満たすWebアプリを開発してください。

基本仕様

  • データ活用： 志賀町のオープンデータ「消防水利」一覧をマッピングする。
  • スマホ最適化： スマートフォンでの利用を想定したレイアウトにする。
  • ロゴ配置： 画面のわかりやすい場所に志賀町の町章ロゴを配置する。
  • ナビゲーション機能： マーカーを押下すると詳細情報が表示され、「ここに行く」ボタンからGoogleマップのナビ機能へリンクする。
  • デザインの統一： トンマナ（トーン＆マナー）は本サイトのCSSカラーを踏襲する。

ディレクトリ構造の指定（練習）

実際の開発現場を想定し、以下のディレクトリ構造でファイルを作成し、HTMLから相対パスで参照するように実装してください。

  • index.html （アプリ本体）
  • ./data/suiri.csv （消防水利のCSVデータ）
  • ./images/logo.png （志賀町の町章ロゴ画像）

利用する素材とヒント

SVGアイコンの利用

マップ上のマーカーは、防火水槽と消火栓で分けて表示します。以下のサイトからアイコンを利用してください。

  • 防火水槽（水滴）：
    https://icooon-mono.com/10938...
  • 消火栓（消火器）：
    https://icooon-mono.com/12869...

Googleマップ連携のヒント

「ここに行く」ボタンのリンク先を以下のように設定することで、スマホのGoogleマップアプリ（ナビ）が直接起動します。

https://www.google.com/maps/dir/?api=1&destination=緯度,経度

AIへの指示（プロンプト）のコツ

「Leaflet.jsを使ってマップを作り、./data/suiri.csv を fetch してマーカーを立てて」
など、要件とディレクトリ構造を具体的にAIに伝えることが成功の鍵です。

UI / UX の検討（アプリの使いやすさ）

要件の最後に「消防水利を必要とするシーンを想定した便利な機能」とあります。AIにコードを作らせる前に、どのようなアプリが使いやすいかアイディアを練りましょう。

検討すべきポイント（例）

  • 起動画面： URLを開いていきなりマップが表示されるのが良いか？ それとも「志賀町 消防水利マップ」という起動画面（スプラッシュ画面）を数秒挟むほうがアプリらしいか？
  • 初期表示： 地図の初期位置は「志賀町役場」にするか？ それとも起動時にGPSを使って「ユーザーの現在地」から表示するか？
  • 検索機能： 土地勘のない人のために、地名や住所で絞り込める検索バーは必要か？
  • 操作性： マップ上に「現在地に戻るボタン」や「防火水槽だけを表示するフィルターボタン」があると便利ではないか？

プロンプトに「起動時に現在地を取得して、そこを中心にマップを描画して」などと追加することで、より実践的なアプリに仕上がります。
